# Astro テーマの選択肢

MkDocsからの移行において、現在の構成（ドキュメント + ブログ）に適したテーマをいくつかピックアップしました。
現在は **Starlight** を使用してセットアップしていますが、好みに応じて他のテーマに変更することも可能です。

## 1. Starlight (現在設定中)
- **特徴**: Astro公式のドキュメント向けテーマ。MkDocs Materialの精神的後継として最適。
- **メリット**:
    - サイドバーナビゲーション、目次、検索機能が標準装備。
    - `docs/` 構造をそのまま維持しやすい。
    - ブログ機能もプラグインやカスタムページで追加可能。
- **デメリット**:
    - "ブログ"というよりは"ドキュメント"の見た目が強い。
- **推奨**: MkDocsの使い勝手を維持したい場合。

## 2. AstroPaper
- **特徴**: ミニマルでSEOに強いブログテーマ。
- **メリット**:
    - 非常に高速でシンプル。
    - タグ、検索、ダークモード完備。
    - 記事中心のデザイン。
- **デメリット**:
    - サイドバーナビゲーションがないため、階層的なドキュメントには不向き。
- **推奨**: ドキュメントよりも「記事」を読ませることに特化したい場合。
- **デモ**: [https://astro-paper.pages.dev/](https://astro-paper.pages.dev/)

## 3. Geeky
- **特徴**: 技術ブログ向けの機能豊富なテーマ。
- **メリット**:
    - グリッドレイアウトやカテゴリ分けが充実。
    - 画像を多用するモダンなブログデザイン。
- **デメリット**:
    - 構造が複雑で、MkDocsからの単純移行だと調整が必要になる可能性が高い。
- **推奨**: メディアサイトのようなリッチな見た目にしたい場合。
- **デモ**: [https://geeky-astro.netlify.app/](https://geeky-astro.netlify.app/)

## 4. MicroFlash
- **特徴**: 非常に軽量なブログテーマ。
- **メリット**:
    - 必要最小限の機能で動作が軽い。
- **デメリット**:
    - 機能が少ないため、カスタマイズが必要になる可能性が高い。

---

## 選択方法

現在は **Starlight** で構築されています。
もし他のテーマ（例: AstroPaper）を試したい場合は、以下の手順で移行（または別ブランチでの構築）を行いますが、ディレクトリ構造の変更が必要になるため、**Starlightのまま進めるか、別テーマを試すか**ご指示いただければと思います。

基本的には、**MkDocsからの移行であればStarlightが最もスムーズ**であり、機能不足を感じない限りはこのまま進めることをお勧めします。
